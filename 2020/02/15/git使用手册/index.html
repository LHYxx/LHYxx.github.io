Git 使用手册

Git分为三个主要区域

  • 工作区，就是我们的本地目录
  • stage，暂存区。
  • master，分支。


图来源 runoob

add

git add test.txt
把目录下的test.txt文件添加到 stage（暂存区）。

commit

git commit -m"describetion"
把暂存区的内容提交到分支。

rm

如果只是简单地从工作目录中手工删除文件，相当于是对工作区进行了修改，可通过add同步到暂存区。运行 git status 时就会在 Changes not staged for commit 的提示。
可以通过再执行add来更新到暂存区。

git rm <file>删除工作区和暂存区的。
相当于先在本地删除<file>，再执行add <file>同步到暂存区。

如果工作区与暂存区不统一，工作区的文件修改过的话，删除的时候需要加参数-f，如下
git rm -f <file>
就将工作区和暂存区都删了。

git rm --cached <file>
删除暂存区的<file>文件，工作区不受影响。

mv

用于重命名、移动
git mv <file_a> <file_b>
将文件a重命名为文件b

reset

reset回退暂存区

git reset HEAD
把暂存区回退到版本库中HEAD所指的版本

git reset HEAD <file>
把暂存区的<file>文件回退到版本库中HEAD所指的版本的<file>文件。

reset回退版本库中HEAD所指的版本

git reset --hard HEAD^
版本库中回退到HEAD所指的上一个版本

git reset --hard 1094a
版本库中回退到 id指定版本

checkout

git checkout .
. 表示整个当前目录。
用暂存区内容替换工作区内容。用来舍弃工作区的修改。
也就是将工作区恢复到最后一次add或者commit的状态。

git checkout -- <file>
指定特定文件，用暂存区<file>替换工作区<file>。用来舍弃工作区<file>文件的修改。
也即是将工作区的<file>文件恢复到最后一次add或commit的状态。

git checkout HEAD .
用HEAD指向的分支中的内容替换暂存区和工作区的内容。也就是将暂存区和工作区恢复到最后一次commit的状态。

git checkout HEAD <file>
用HEAD指向的分支中的文件替换暂存区和工作区的<file>文件。也就是将暂存区和工作区的<file>文件恢复到最后一次commit的状态。

clone

git clone <url>
git clone <url> <directory>
使用 git clone 从Git 仓库中拷贝项目。
例如从Github上将项目拷贝到本地，例如
git clone https://github.com/example/example
可以通过<directory>参数指定本地保存目录

push

git push <url>
修改后的文件上传到远程仓库。

分支管理

使用分支将工作切分开来，从而让我们能够在不同开发环境中做事，并来回切换。

git branch <name>
创建一个分支，名称为<name>

git checkout <branch name>
切换到名为<branch name>的分支

git checkout -b <branch name>
创建并切换到<branch name>分支中

git branch -d <branchname>
删除分支

git merge <branch name>
分支合并。将<branch name>分支合并到主分支。必须在master中执行该命令。
当某一文件在分支与主分支之间内容不同时，合并会产生冲突。必须手动解决冲突之后，再进行合并。将文件修改为我们希望的内容，然后通过add 和 commit提交之后，再进行合并。

其他

git init
在一个目录下执行git init，创建仓库。把该目录变为git仓库

git diff HEAD -- test.txt
查看版本库中最新版本和工作区中的test.txt文件的区别

git log
git log --pretty=oneline
查看提交记录

git reflog
查看命令记录
